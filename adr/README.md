# Architecture Decision Records

Short, durable notes on decisions that shape Lodestar's architecture — and, just as importantly,
*why* they were made and what was traded away.

Write one when a decision is hard to reverse, surprising, or likely to be questioned later
(e.g. "vault-as-database", "queue tasks are pipeline steps", "secrets only in the OS keychain").

**Convention:** one file per decision, numbered in order, e.g. `0001-vault-as-source-of-truth.md`.
Keep each brief — context, the decision, the consequences.

This README also keeps the otherwise-empty folder in git.
